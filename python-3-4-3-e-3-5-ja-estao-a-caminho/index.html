Python 3.4.3 e 3.5 já estão a caminho

Alexandre Vicenzi
09/02/2015 · 2 minutos de leitura

Atenção

Este artigo foi migrado do Wordpress e pode conter erros de formatação.

Caso encontre algum problema deixe um comentário que iremos resolver o mais breve possível.

Ontem (08/02), foi liberado o Python 3.4.3rc1 e o 3.5.0a1. A versão 3.4.3 está prevista para dia 22/02 e a 3.5 para 13/09.

A versão 3.4.3rc1 vem com várias correções de bugs e pequenas melhorias em relação a 3.4.2. Já a versão 3.5, ainda está em desenvolvimento, e a 3.5.0a1 é a primeira liberação desta nova versão. Várias alterações ainda estão sendo planejadas para a 3.5.

As principais novidades para a 3.4.3rc1 são

  • PEP 428, Novo módulo (pathlib) para acesso ao sistemas de arquivos
  • PEP 435, Novo módulo (enum) padronizado para enumerations
  • PEP 445, Uma nova API C para implementação de alocação de memória customizada
  • PEP 450, Novo módulo (statistics) para estatística.
  • PEP 453, Um instalador para o pip (package manager)
  • PEP 454, Novo módulo (tracemalloc) para rastreamento de alocação de memória.
  • PEP 456, Novo algoritmo de hash para strings e binary data
  • PEP 3154, Protocolo novo e melhorado para objetos pickled
  • PEP 3156, Novo módulo (asyncio) para I/O assíncrono, eventos, corotinas e tarefas.
Para conferir a lista completa de alterações desta versão clique aqui.

As principais novidades para a 3.5.0a1 são (até o momento)

  • PEP 461, Adicionar suporte a "%-formatting" para bytes e array de bytes
  • PEP 465, Novo operador (@) para multiplicação de matriz
  • PEP 475, Adicionar suporte a novas tentativas automáticas após interrupções do sistema
Para conferir a lista completa de alterações desta versão clique aqui.

Assine nosso feed e não perca nenhuma novidade sobre estas novas versões do Python.
